# BetterModifiers progress

Append-only milestone log.

## Milestones

- [x] **M0 — Migration scaffold.** New `mactools/bettermodifiers/` SwiftPM package, `Package.swift` with `LayerKeyHID` C target + `bettermodifiers` executable, `.gitignore`, `.agent/` skeleton, ad-hoc bundle build verified with `swift build`.
- [x] **M1 — Engine port.** `EventTapController`, `CapsLockController`, `PermissionsController`, and the `LayerKeyHID` C bridge ported into `Sources/bettermodifiers/Engine` and `Sources/LayerKeyHID`. Tab + Caps layer behavior and the synthetic `flagsChanged` browser fix preserved.
- [x] **M2 — Rule model + persistence.** `Trigger`, `ModifierMask`, `Rule`, `RulesStore` (JSON in `~/Library/Application Support/BetterModifiers/rules.json`, debounced atomic writes, O(1) lookup cache). Engine consumes rules via `(trigger, inputKey)` lookup. Default seed = LayerKey MVP (`Tab + 0..9 -> Option + 0..9`).
- [x] **M3 — SwiftUI window.** `MainWindow` (NavigationSplitView), `RulesView`, `RuleEditorView`, `KeyRecorderView`, `KeyChip`/`KeyComboView`, `GeneralView`, `AboutView`. Rule list grouped by trigger, edit/delete/toggle, live preview, conflict warning.
- [x] **M4 — Menu bar + launch at login.** `MenuBarController` owns the `NSStatusItem`, exposes Status / Enable / Open window / Launch at Login / Accessibility / Open Accessibility Settings / Quit. `LaunchAtLoginController` uses `SMAppService.mainApp`.
- [x] **M5 — Legacy LayerKey migrator.** `LegacyMigrator.runOnce()` (guarded by `UserDefaults`) removes legacy LaunchAgent plist and `~/Applications/LayerKey.app` / `~/Applications/ModifierOverride.app`.
- [x] **M6 — Repo docs + install-all integration.** Root `README.md` table, `scripts/install-all.sh`, `scripts/build-install-local.sh`, and tool docs updated. `mactools/layerkey/` removed.
- [x] **M7 — LayerKey purge + HID rename.** `LayerKeyHID` C target renamed to `BetterModifiersHID` (folder, headers, include guard `BETTER_MODIFIERS_HID_H`, functions `bm_*`). `Package.swift` and `CapsLockController` updated.
- [x] **M8 — Explicit modifier toggles in rule editor.** New `ModifierTogglesView` (⌃⌥⇧⌘ chip toggles). `RuleEditorView` output row split into `ModifierTogglesView` + key-only `KeyRecorderView`; `keyWithModifiers` mode removed so system shortcuts like `⌃⌘ + Arrow` can be authored without the OS swallowing them.
- [x] **M9 — Hyper Key page + engine integration.** New `AppSettings` + `SettingsStore` (JSON in `~/Library/Application Support/BetterModifiers/settings.json`, same `@Sendable` debounced save pattern as `RulesStore`). `EventTapController` now bypasses per-rule lookup and emits `inputKey + hyper.modifiers` whenever a trigger is in hyper mode. `HyperKeyView` cards expose toggle + modifiers + live preview. `RulesView` shows a banner and dims rules when hyper is on.
- [x] **M10 — Sidebar reorder + per-trigger Rules cards + page descriptions.** Sidebar order is now Hyper Key → Rules → General → Appearance → About. Each page starts with a `PageHeader` (title + subtitle). `RulesView` is a scroll of two `GroupBox` cards (Tab / Caps Lock) with proper row padding so edit/delete sit inset from the right edge.
- [x] **M11 — Appearance section.** `AppearanceView` adds a System / Light / Dark theme picker (applied via `NSApp.appearance`) and a "Hide menu bar icon" toggle. `MenuBarController.setVisible` removes/recreates the `NSStatusItem`. `AppDelegate.applicationShouldHandleReopen` reopens the main window from Finder so users are not stranded when the icon is hidden.
- [x] **M12 — Sparkle 2 updates.** Added Sparkle SPM dependency, `Info.plist` keys (`SUFeedURL`, `SUPublicEDKey`, `SUEnableAutomaticChecks`, `SUScheduledCheckInterval`, `SUAllowsAutomaticUpdates`), `UpdateController` wrapping `SPUStandardUpdaterController`, and About page Updates row (auto-check toggle, Check Now button, last-checked text). Build script embeds and signs `Sparkle.framework` under `Contents/Frameworks`.
- [x] **M14 — 1.0: SwiftUI Scene window + clean title bar + non-quitting close + responsive Rules layout.** Replaced the hand-rolled `NSWindow` + `NSHostingController` setup with a SwiftUI `App` + `Window("BetterModifiers", id: "main")` Scene (`@NSApplicationDelegateAdaptor` keeps the existing `AppDelegate`). This is the only configuration in which `.toolbar(removing: .sidebarToggle)` actually reaches the underlying `NSSplitViewController`, so the duplicate trailing toggle is finally gone. A single custom `ToolbarItem(placement: .navigation)` button lives in the leading slot with the standard `.unified` toolbar metrics (proper size + padding). Sidebar toggle animates via `withAnimation(.easeInOut(0.25))` inside `SidebarVisibility.toggle()`. `applicationShouldTerminateAfterLastWindowClosed` returns `false` so closing the window keeps the engine running in the menu bar; the global `NSWindow.willCloseNotification` observer in `AppDelegate` then runs `setActivationPolicy(.accessory)` + `NSApp.hide(nil)` to drop the Dock tile. `RulesView` wraps the trigger cards in a `GeometryReader` + `ScrollView([.vertical, .horizontal])`: each card is pinned to at least the visible detail-pane width (so cards stretch to the left edge like General/Appearance) and grows beyond it on narrow windows so the outer scroll surfaces the right-most chip + delete button. Inside each row, the chip cluster has `.fixedSize(horizontal: true)` and is sandwiched between leading/trailing `Spacer(minLength: 0)`s, so the cluster reads as centered when there's slack and stays at its readable minimum width when there isn't. Bumped to `1.0.0` (build `100`).
- [x] **M16 — Shift+Space order-independence + composable extra modifiers + plain-Space safety.** Reworked `ShiftSpaceState` to *always* arm on the first Space-down, but forward the original Space key-down whenever Shift wasn't held yet (`forwardedSpaceDown` flag). That preserves plain typing, Cmd+Space (Spotlight), and hold-to-repeat completely; auto-repeat key-downs are dispatched based on the same flag (forward when forwarded, swallow when pre-armed). Added `handleFlagsChanged` which posts a synthetic Backspace (`KeyCodes.delete = 51`) and clears `forwardedSpaceDown` when Shift comes in cleanly (no Cmd/Ctrl/Opt) while a forwarded Space is still held - the standard AutoHotkey "delete the just-typed space and slide into layer mode" trick. Layer dispatch now keys off `event.flags.contains(.maskShift)` regardless of arrival order, and `composableExtraFlags(_:)` ORs any held Cmd/Ctrl/Opt into the rule's output flags so chords like `⇧Space + ⌘ + J` fire `⌘ + (rule output) + J` without being blocked. Space-up dispatch is now tri-state: forwarded → forward the matching key-up; not forwarded + used as layer → swallow; not forwarded + not used → emit synthetic `Shift+Space` chord (apps that bind that chord still see it). Unmapped third keys with Shift held fall through normally instead of being marked as layer use, so the user still gets the fallback chord on release.
- [x] **M15 — Shift+Space chord trigger.** New `Trigger.shiftSpace` case (raw value `shiftSpace`, displayName "Shift + Space", chipLabel "⇧Space", SF Symbol `space`). `KeyCodes.space = 49`. `EventTapController` gained `ShiftSpaceState` (mirrors `TabState`) plus `isShiftSpaceLayerTrigger(event:)` and `hasNonShiftUserModifiers(_:)` helpers: Space-down is only swallowed when Shift is held AND no other user modifier is held, so plain Space typing and Cmd/Ctrl/Opt+Space shortcuts pass through untouched. While the layer is armed, Shift acts purely as the trigger qualifier — emitted events use `rule.outputModifiers.eventFlags` directly so Shift never leaks unless a rule asks for it. If no third key arrives before Space-up, a synthetic `Shift+Space` chord is posted so chord typing is preserved; if a third key was Shift+Space-chorded but didn't match any rule (and Modifier Mode is off), the state is marked `usedAsLayer` so we don't double-post the fallback. `AppSettings` seeds a `.shiftSpace` Modifier-Mode entry (`[.command, .option, .control]` — Shift omitted to avoid noise). `RulesView` now reads `trigger.symbolName` instead of a hardcoded Tab/Caps switch, so the new card renders with the spacebar glyph. Cards/Modifier-Mode entries appear automatically via `Trigger.allCases`.
- [x] **M13 — Inline rule editing + accessibility self-healing + theme polish.** Replaced the `RuleEditorView` sheet with an `InlineRuleRow` component: tap any chip to edit (modifier toggles flip in place; tapping an input/output key chip starts `NSEvent` recording, the next non-modifier key commits via `RulesStore.update`). "Add rule for X" appends a new row pre-set to the first unused digit and auto-enters input recording. Bigger circular delete button, hover highlight, `28 pt` row padding, centered chip cluster. `MainWindow` now hides the title-bar sidebar-toggle (`.toolbar(removing: .sidebarToggle)` on macOS 14.4+) and tints detail panes with `underPageBackgroundColor` so the light theme reads as System Settings grey. `AppDelegate` prompts for Accessibility on first launch via `AXIsProcessTrustedWithOptions`. `EventTapController` runs a 4 s health check after `tapCreate`; if no events arrive (TCC silently revoked us after an ad-hoc rebuild) it flips status to `tapNotReceiving` and re-prompts. New "Restart Engine" item in the menu bar and `GeneralView` lets the user re-create the tap on demand. SwiftPM platform bumped to macOS 14. `RuleEditorView.swift` deleted.
